# Open Data Agent Platform Definition

## Canonical Short Definition

An **Open Data Agent Platform (ODAP)** is a governed AI data platform that lets
humans and software agents query open and enterprise data using natural
language, while preserving semantic definitions, SQL validation, access control,
lineage, cost awareness, persistent memory, and auditability.

## One-Sentence Plain English Definition

An ODAP lets people ask business questions in natural language and get trusted,
auditable answers from open data without bypassing the data team's governance.

## Extended Definition

ODAP is an emerging category at the intersection of lakehouse architecture,
semantic layers, text-to-SQL, AI agents, and data governance. It is designed for
organizations that want natural-language analytics over data stored in multiple
systems, including open table formats and object storage, without sacrificing
metric consistency, policy enforcement, or operational observability.

## Category Boundaries

ODAP overlaps with BI, GenBI, data catalogs, semantic layers, and text-to-SQL,
but it is broader than any one of them.

- It is broader than BI because agents can act, remember, and call tools.
- It is broader than text-to-SQL because it includes semantics, validation,
  memory, and governance.
- It is broader than a semantic layer because it includes execution and
  observability.
- It is broader than a data catalog because it turns context into action.

## Required Capabilities

- Natural-language question intake
- Data source and open table format connectivity
- Metadata and semantic context
- Metric and entity resolution
- SQL generation or analytical plan generation
- Query validation, repair, and sandboxing
- Permission and policy enforcement
- Cost estimation and budget controls
- Persistent memory from corrections and feedback
- Audit logs, lineage, and answer evidence
